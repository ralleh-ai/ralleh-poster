# Few-Shot Example Template (Required)

Use this structure for every example file in `examples/`.

## Example Metadata
- Outcome: Success | Failure
- Example Type: Positive | Negative
- Primary Stage Checkpoint: Stage X
- Primary Rules Referenced: list of `STANDARDS.md` sections

## 1) Event Brief
- Title
- Date/Time
- Venue
- Genre
- Style Preference

## 2) Candidate Strategy / Prompt
Include exact strategy summary and the prompt used.

## 3) Critique Mapping
Map findings to explicit standards/workflow references.

## 4) Final Result
- Success examples: why it passed
- Failure examples: why it failed

## 5) Lesson Captured
One operational lesson that should generalize to future runs.

## 6) Technical Manifest
- Model
- Size/aspect
- Seed (if available)
- Output format
